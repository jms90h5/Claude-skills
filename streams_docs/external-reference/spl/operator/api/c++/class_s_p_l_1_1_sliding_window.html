Detailed Description

template<class T, class G = int32_t, class D = std::deque<T>, class S = std::unordered_map<G,D>>
class SPL::SlidingWindow< T, G, D, S >

Class that represents a sliding window.

Example: SlidingWindow<TupleType, PartitionType, StorageType>

TupleType is the type of the tuples stored in the window (can be a pointer type or a non-pointer type). PartitionType, which is optional, is the type of the window partition specifier, and the StorageType, which is optional, is the internal data structure used for storing tuples (which maps partitions to a sequence of tuples e.g.: unordered_map<PartitionType, vector<TupleType> >).

A sliding window operates by collecting tuples until a full window is reached, and then maintaining the full window at all times as new tuples are inserted, by evicting the oldest tuples from the window.

For a detailed description of operational semantics for sliding windows, see window policy classes used to configure them: CountWindowPolicy, DeltaWindowPolicy, TimeWindowPolicy.

Sliding widows allow registering event handlers for the following events defined in WindowEvent:

  • beforeTupleInsertionEvent
  • afterTupleInsertionEvent
  • beforeTupleEvictionEvent
  • afterTupleEvictionEvent
  • windowTriggerEvent
  • windowInitialFullEvent
  • windoPartitionEvictionSelection
  • windowPartitionEviction
