Detailed Description

template<typename T, typename G = int32_t, typename D = std::deque<T>, typename S = std::unordered_map<G,D>>
class SPL::TimeIntervalWindowPane< T, G, D, S >

Window which stores tuples with an event time which falls within a specified interval.

This window stores tuples with event time values greater or equal to the window start and less than the window end. The client does not directly instantiate this class; instead, it creates an instance of TimeIntervalWindow, which manages and assigns tuples to a set of window time intervals. For more details see the TimeIntervalWindowOptions class.

Template Parameters
T type of the tuples stored in the window. The SPL operators which use event time windows use std:shared_ptr<Tuple>, so the system does not have to create separate copies for tuples which are assigned to overlapping window time intervals.
G type of the window partition specifier - the default value is int32_t
D type of the tuple collection container - the default value is std::deque<T>
S internal data structure used for storing tuples, which maps partitions to the tuple collection container - the default value is std::unordered_map<G, D>.

An event-time window time interval operates by collecting tuples as they arrive. When the trigger fires, the operator performs its processing against the content of the window.

A TimeIntervalWindowPane has a state which can take one of the following values:

  • Incomplete - the latest inserted SPL::Watermark has a value which is less than the window's end time .
  • Complete - the latest inserted SPL::Watermark has a value which is greater than or equal to the window's end time but less than the end time plus the maximum tuple lateness as defined by the TimeIntervalWindowOptions used for configuring the parent window.
  • Closed - the latest inserted SPL::Watermark is greater than the end time plus the maximum tuple lateness.

TimeIntervalWindowPane windows can be partitioned. A partition is a set of tuples that have the same values for the partitioning attributes.

A TimeIntervalWindowPane is triggered by default when one of the following conditions is met:

  • a Watermark is inserted with a value greater or equal to the window's end time . The window's state transitions from State::Incomplete to State::Complete before the window triggers. A trigger event is emitted for each window partition.
  • a Watermark is inserted after one or more late events are inserted into the window (late events are those arriving after the window becomes Complete). Trigger events are emitted only for the updated partitions.

A TimeIntervalWindowPane fires the following events :

  • beforeTupleInsertionEvent - fired before a tuple is inserted into the window time interval
  • afterTupleInsertionEvent - fired after a tuple is inserted into the window time interval
  • windowInitialFullEvent - fired immediately after the window's state transitions from Incomplete to Complete
  • windowTriggerEvent - fired every time when when the window triggers
  • beforeWindowCloseEvent - fired right after the window's state transitions from Complete to Closed
