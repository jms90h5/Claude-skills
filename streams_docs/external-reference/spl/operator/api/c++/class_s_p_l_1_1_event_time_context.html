Detailed Description

Event-time optional context interface for an SPL Operator.

The context provides functions for event-time related processing. The client acquires the context using the EventTimeContext name when the current operator is within the reachability graph of an operator annotated as an event source.

Since
IBM Streams Version 4.3.0

Member Enumeration Documentation

◆ WatermarkMode

enum SPL::EventTimeContext::WatermarkMode

Event-time watermark processing modes.

Enumerator
inputWatermark 

The operator's watermark is set by the Streams runtime as the minimum of watermark values received on all input streams.
outputEventTime 

The operator's watermark is set by the Streams runtime based on the event time of submitted tuples.

Watermarks are generated assuming that tuples may be submitted out of order, but only up to a certain interval. After a tuple with event-time N the operator may submit a tuple with an event-time greater than or equal to N - lag, where lag is defined by the current SPL annotation. In order for such tuples to not be considered late, the runtime sets the operator's watermark as the maximum event-time of submitted tuples minus lag.
manual 

The operator's watermark is set by calling setWatermark(). A new watermark is set only if it is greater than the current operator watermark, otherwise it is ignored.
