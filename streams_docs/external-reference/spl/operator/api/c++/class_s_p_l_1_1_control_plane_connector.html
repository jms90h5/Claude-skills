Detailed Description

Used for establishing a connection to the Job Control Plane (JCP).

The initial state of an object of this type is unconnected. Clients must call its connect method to establish the connection. Clients can subscribe to notifications emitted by a connector which reflect changes in the connection state.

Since
InfoSphere® Streams Version 4.0.0.

Adds a listener which will be informed of connection status changes.

The listener will receive notifications of type The same listener can be added more than once with the same or different values for the filter and handback. There is no special treatment of a duplicate entry. For example, if a listener is registered twice with no filter, then its handleNotification method will be called twice for each notification. listener The listener object which handles the notifications. filter Pointer to a filter object provided by the listener. If NULL, no filtering will be performed before handling notifications (all notifications are to be delivered). handback Pointer to be sent back to the listener when a notification is emitted. The pointer is not used by the ControlPlaneConnector instance; instead it is resent unchanged back to the listener with the notification. #removeConnectionNotificationListener

Removes a connection status notification listener.

A listener must be have been registered that exactly matches the given filter, and handback parameters. If there is more than one such listener entry in this connector's list, only one is removed.

The filter and handback parameters may be NULL if and only if they are NULL in a listener to be removed.

Parameters
listener A listener that was previously added to this object.
filter The filter that was specified when the listener was added.
handback The handback that was specified when the listener was added.
Exceptions
SPLListenerNotFoundException The listener is not registered with this object, or it is not registered with the given filter and handback.
