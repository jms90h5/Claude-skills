Detailed Description

Used for communicating with the Job Control Plane (JCP).

It contains the functions necessary to register or unregister a listener for handling Notifications emitted by JMX MBeans registered with JCP, access MBean attributes, and invoke operations.

MBeans are specified using the canonical form of their ObjectName. More precisely, the canonical form is a string consisting of the domain part, a colon (:), and the canonical key property list.

The canonical key property list is a string representation of the list of key properties, in which the key properties are sorted in lexical order, where the comparison is based on the Unicode value of each character in the strings. See the javax.management.ObjectName Java API documentation for further details.

ObjectName instances containing pattern indications are not supported.

For more information about the canonical form of an ObjectName, see the documentation of the Java Management Extensions (JMX ™) API.

Since
InfoSphere® Streams Version 4.0.0.

Adds a notification listener.

The listener will receive notifications of type The same listener can be added more than once with the same or different values for the filter and handback. There is no special treatment of a duplicate entry. For example, if a listener is registered twice with no filter, then its handleNotification method will be called twice for each notification. source The canonical form (see ControlPlaneConnection detailed description ) of the JMX ObjectName associated with the notification source.

Parameters
listener The listener object which handles the notifications.
filter Pointer to a filter object provided by the listener. If NULL, no filtering will be performed before handling notifications (all notifications are to be delivered).
handback Pointer to be sent back to the listener when a notification is emitted. The pointer is not used by the ControlPlaneConnection instance; instead it is resent unchanged back to the listener with the notification.
Exceptions
SPLControlBeanNotFoundException The source name provided does not match any of the MBeans registered with the JCP.
SPLRuntimeIOException A communication failure occurred when talking to the JCP.

Removes a notification listener.

A listener must be registered that exactly matches the given filter, and handback parameters. If there is more than one such listener, only one is removed.

The filter and handback parameters may be NULL if and only if they are NULL in a listener to be removed.

Parameters
source The canonical form (see detailed description ) of the JMX ObjectName associated with the notification source.
listener A listener that was previously added to this object.
filter The filter that was specified when the listener was added.
handback The handback that was specified when the listener was added.
Exceptions
SPLControlBeanNotFoundException The source name provided does not match any of the MBeans registered with the JCP.
SPLRuntimeIOException A communication failure occurred when talking to the JCP.
SPLListenerNotFoundException The listener is not registered with this object, or it is not registered with the given filter and handback.

Invokes an operation on an MBean registered with the Job Control Plane.

For example, suppose you have the following MBean interface:

public interface FooMBean {
    public String bar(String arg1, int arg2);
}

The bar operation can be invoked as follows:

     ControlPlaneConnection conn; // obtained from connector

     rstring arg1("value1");
     int32 arg2 = 2;

     // argument values
     vector<string> args;
     args.push_back(rstring::toString(arg1));
     args.push_back(rstring::toString(arg2));

     // argument types
     vector<SPL::Meta::Type::Value> types;
     types.push_back(SPL::Meta::Type::RSTRING);
     types.push_back(SPL::Meta::Type::INT32);

     string retVal = conn->invoke("com.ibm.streams.control:type=foo", "bar", &args, &types);
Parameters
name The canonical form (see detailed description ) of the JMX ObjectName associated with the notification source.
operationName The name of the operation to be invoked.
args pointer to vector of arguments passed to the method invocation. Each vector element is a string representation of the parameter value. If no arguments are passed, then the pointer may be NULL or point to an empty vector.
argTypes pointer to array containing the argument types represented as SPL::Meta::Type::Value. If no arguments are passed, then this argument is ignored.
Returns
The result of invoking the operation on the specified Job Control Plane MBean, serialized as string.
Exceptions
SPLControlBeanNotFoundException The provided MBean name does not match any of the MBeans registered with the JCP.
SPLRuntimeIOException A communication failure occurred when talking to the JCP.
SPLControlBeanException Indicates an exception thrown by the control MBean.
