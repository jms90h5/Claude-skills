    Administer the product by using the Teracloud® Streams graphical user interface, APIs, or the streamtool command-line interface.

    • Domains

      After you create a Teracloud® Streams domain, you can administer it by using the Streams Console graphical user interface and the streamtool command-line interface.

    • Resources

      After you add resources to domains and instances, you can administer them by using the Streams Console or the streamtool command-line interface.

    • Instances

      To create and manage instances, you can use the Streams Console or the streamtool command-line interface.

    • Domain and instance services

      Before you can perform Teracloud® Streams tasks, Teracloud Streams domain and instance services need to be running on resources in the domain. Domain services are started by starting the domain and instance services are started by starting the instance.

    • Jobs

      You can start and administer jobs by using Streams Console or the streamtool command-line interface.

    • Recovery

      You can back up your Teracloud® Streams system files and directories and then restore them later if necessary. In general, you can recover from application and runtime failures without restoring Teracloud Streams from a backup.

      • Backing up your Teracloud® Streams environment

        After you install Teracloud® Streams, use the information in this section to back up and restore your system. By saving a snapshot of your Teracloud Streams environment, you can restore your system to the same environment if the system crashes.

      • Recovering from Teracloud® Streams domain and instance failures

        If your domain or instance is not functioning well because of failed hardware or software components, Teracloud® Streams can automatically fail over to standby management services to provide high availability.

    • Streams Console

      You can use the Streams Console to monitor and manage the Teracloud® Streams instances and applications in your domain from any computer that has HTTP connectivity to the server that is running the web management service.

Backing up, restoring, and recovering Teracloud® Streams

You can back up your Teracloud® Streams system files and directories and then restore them later if necessary. In general, you can recover from application and runtime failures without restoring Teracloud® Streams from a backup.

  • Backing up your Teracloud Streams environment
    After you install Teracloud® Streams, use the information in this section to back up and restore your system. By saving a snapshot of your Teracloud® Streams environment, you can restore your system to the same environment if the system crashes.
  • Recovering from Teracloud Streams domain and instance failures
    If your domain or instance is not functioning well because of failed hardware or software components, Teracloud® Streams can automatically fail over to standby management services to provide high availability.
