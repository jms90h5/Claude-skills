  • Administering

    Administer the product by using the Teracloud® Streams graphical user interface, APIs, or the streamtool command-line interface.

    • Domains

      After you create a Teracloud® Streams domain, you can administer it by using the Streams Console graphical user interface and the streamtool command-line interface.

    • Resources

      After you add resources to domains and instances, you can administer them by using the Streams Console or the streamtool command-line interface.

    • Instances

      To create and manage instances, you can use the Streams Console or the streamtool command-line interface.

      • Instance identifiers

        An instance name or identifier is the name that you specify when you create an instance.

      • Creating instances

        Use the Streams Console or the streamtool mkinstance command to create a Teracloud® Streams instance. When you create an instance you specify the instance name and properties and the resources that run the instance.

        • Resource specification options

          The Streams Console and the streamtool mkinstance command provide several options for specifying the static resources in a Teracloud® Streams instance.

        • Creating a host specification file from compiled applications

          You can use the streamtool mkhostfile command to generate a host file that is based on the host and host pool specifications in a stream application.

      • Starting and stopping instances

        To run Teracloud® Streams, you must start a Teracloud Streams instance.

      • Monitoring instances

        You can monitor the status and health of your Teracloud® Streams environment and capture the information for offline viewing and analysis.

      • Updating instance properties

        After you create a Teracloud® Streams instance, you can configure it by updating its properties.

      • Removing instances

        When you remove a Teracloud® Streams instance, the instance record in the configuration repository is removed along with all persistent instance state information.

    • Domain and instance services

      Before you can perform Teracloud® Streams tasks, Teracloud Streams domain and instance services need to be running on resources in the domain. Domain services are started by starting the domain and instance services are started by starting the instance.

    • Jobs

      You can start and administer jobs by using Streams Console or the streamtool command-line interface.

    • Recovery

      You can back up your Teracloud® Streams system files and directories and then restore them later if necessary. In general, you can recover from application and runtime failures without restoring Teracloud Streams from a backup.

    • Streams Console

      You can use the Streams Console to monitor and manage the Teracloud® Streams instances and applications in your domain from any computer that has HTTP connectivity to the server that is running the web management service.

    • Logs and traces

      Teracloud® Streams provides options that enable administrators, instance owners, and application developers to manage log and trace data.

    • Managing and monitoring Teracloud® Streams using management APIs

      Teracloud® Streams provides two API sets to manage and monitor Streams services and jobs.

Creating instances

Use the Streams Console or the streamtool mkinstance command to create a Teracloud® Streams instance. When you create an instance you specify the instance name and properties and the resources that run the instance.

Procedure

  • To create an instance in a basic domain, see Creating a Teracloud® Streams basic domain and instance.
  • To create an instance in an enterprise domain, see Creating a Teracloud® Streams instance in an enterprise domain.
  • To configure an instance to use dynamic resource allocation, see Configuring an instance to use dynamic resource allocation.

What to do next
