        • Operating system requirements for Teracloud® Streams

          Teracloud® Streams supports Red Hat Enterprise Linux™ (RHEL).

        • Required RPMs for Teracloud® Streams

          To operate correctly, Teracloud® Streams requires a set of prerequisite RPMs.

          • RPM minimum versions and availability

            For most RPMs, the default operating system version meets the Teracloud® Streams requirements. The RPMs that are required for Teracloud Streams are available in the operating system installation package.

          • Installing RPMs for Teracloud® Streams

            You can install the required RPMs for Teracloud® Streams by using the dnf install command that is included with Red Hat Enterprise Linux™ (RHEL).

        • Java requirements, options, and settings

          Ensure that you use a Java™ development kit that is supported by Teracloud® Streams. If Java memory issues occur, Teracloud Streams provides configuration properties that you can use to allocate more memory for the Java environment.

        • Dependency checker script

          The dependency checker script verifies that your system satisfies the operating system, RPM, and other software requirements for Teracloud® Streams.

      • Domain controller service options

        A domain controller service runs on every resource in a Teracloud® Streams domain and manages all of the other services on that resource. You can set up the domain controller service as a registered Linux™ system service or an unregistered service.

      • Firewall configuration guidelines

        If firewall usage is required, the preferred configuration is to set up a firewall at the perimeter of the Teracloud® Streams cluster to restrict network access to resources in the cluster but not communication between the resources. When any communication passes through a firewall, latency is introduced. These guidelines apply to clients that are in the Teracloud Streams cluster as well as clients that are external to the cluster.

      • Linux ulimit configuration guidelines

        The default Linux™ user limit (ulimit) values might be too small for some Teracloud® Streams environments. Use these guidelines to ensure that your ulimit settings are sufficient for Teracloud Streams.

      • Name resolution requirements for resources

        All resources must satisfy the product network configuration requirements or Teracloud® Streams fails to start on the resource.

RPM minimum versions and availability for Teracloud® Streams

For most RPMs, the default operating system version meets the Teracloud® Streams requirements. The RPMs that are required for Teracloud® Streams are available in the operating system installation package.
Notes:
  • Teracloud® Streams is tested with the minimum RPM versions that are listed in the following tables. Other RPM versions might work with the product, but they have not been tested.
  • Running the Teracloud® Streams dependency checker script verifies that all the required RPMs are installed.
  • In addition to the RPMs that are required for the Teracloud® Streams product, there are RPM requirements for several of the SPL specialized toolkits. The dependency checker script does not identify RPMs that are required for these toolkits. For more information, see Required RPMs for the Teracloud Streams specialized toolkits.
For more detailed information about required RPMs for the Teracloud® Streams product, see the table for your operating system version and hardware system:

    • Required RPMs for RHEL 8.10 and RHEL 9.4 on x86_64 systems

  2. For the openssl RPM, Streams requires a minimum version of 1.0.1e.
