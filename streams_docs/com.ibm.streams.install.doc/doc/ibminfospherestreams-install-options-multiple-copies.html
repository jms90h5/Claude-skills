Multiple installations of Teracloud® Streams on the same resource

To run multiple installations of Teracloud® Streams on the same resource, each installation must be installed in a unique location.

You can access multiple installations of Teracloud® Streams at the same time by using the streamsprofile.sh script. This script determines which installation of Teracloud® Streams you are using.

When you run the source product-installation-root-directory/7.2.0.0/bin/streamsprofile.sh command, your shell session uses that installation of Teracloud® Streams. To use another installation, you can source a different streamsprofile.sh script from your shell session.

If you are using a shell initialization file such as home-directory/.bashrc to source the streamsprofile.sh script, you can change the default installation that you use by editing the initialization file before you run the source command.

The configuration files for a Teracloud® Streams installation are stored in the installation-owner-home-directory/.streams directory. Included in this directory are license files, global configuration data, and instance data. Be careful with the global configuration data because its location is shared by all the Teracloud® Streams installations.

Installing a later version of Teracloud® Streams in the same location as a previous installation upgrades only that particular installation. No other installations on the same resource are affected.

Uninstalling Teracloud® Streams only affects that particular installation. This action does not affect other installations on the same resource.
